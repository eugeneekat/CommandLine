Консольное приложение коммандная строка на С++

Структура файлов и общая логика взаимодействия классов:

Interpritator.h - Главный объект Interpritator
Interpritator принимает пользовательский ввод строки которая является
коммандой выполнения для Interpritator.

Интерпритатор содержит в себе 3 модуля:
MyParser.h - Объект MyParser
MyLogger.h - Объект MyLogger
DiscManager.h - Объект DiscManager

MyParser - является наследником абстрактного класса IParser который
находится в файле IParser.h. IParser определяет метод парсинга.

MyParser получает строку пользовательского ввода от Interpritator и 
реализует стратегию парсинга пользовательского ввода на отдельные
составляющие(команда и аргументы) которые потом возвращает обратно
классу Interpritator.

DiscManager является наследником абстрактного класса IExecutable
который находится в файле IExecutable.h. IExecutable определяет метод
выполнения команд.

DiscManager получает набор лексем от Interpritator которые предварительно
было обработаны парсером и на их основе выполняет уже конечную команду,
либо выбрасывает определенное исключение (если команды нету, либо по тем
или иным причинам команда не может быть выполнена)

MyLogger - является наследником абстрактного класса ILogger который
находится в файле ILogger.h. ILogger определяет метод записи ошибок

MyLogger - перехватывает все ошибки выборшенные DiscManager и записывает
в файл MyLogger.log.

MyException.h - реализация моего класса исключения

exam.cpp - метод Main()
